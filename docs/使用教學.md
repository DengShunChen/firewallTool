# fwctl 使用教學

本文件說明 **firewalld** 的管理邏輯，以及 **fwctl** 的建議用法、權限與 **direct** 等進階功能的操作步驟。適合已安裝 `firewalld`、且熟悉基本 Linux／網路管理的人閱讀。

---

## 1. fwctl 是什麼？

- **fwctl** 是包在 **`firewall-cmd`**（線上、經 D-Bus）外層的 CLI，用較清楚的輸出與子命令整理常見操作。
- 可選 **`fwctl --offline …`** 改呼叫 **`firewall-offline-cmd`**，在 **daemon 未執行**時改磁碟上的設定（見第 9 節）。
- 它**不取代** `firewall-cmd` 的全部選項；複雜情境仍可查 `man firewall-cmd` 後自行下指令。

---

## 2. firewalld 標準管理邏輯（必讀）

### 2.1 Zone（區域）優先

- 連線／介面會掛在 **zone**（如 `public`、`trusted`）。
- **預設 zone** 影響「沒特別指定時」新連線落在哪。
- 多數規則表達成：**在某 zone 允許哪些 service／port／rich rule**。

### 2.2 Runtime 與 Permanent（兩套設定）

| 類型 | 說明 |
|------|------|
| **Runtime** | 立刻作用在 **daemon 記憶體**；未寫入 permanent 時，重啟 `firewalld` 或開機後可能不同。 |
| **Permanent** | 寫入 **設定檔**（`/etc/firewalld/…`）；**不會自動**讓目前線上狀態變一樣。 |

讓「線上」與「永久檔」一致，標準作法是：

```bash
sudo "$(command -v fwctl)" reload --yes
# 等同 firewall-cmd --reload
```

### 2.3 規則層級（由建議到進階）

1. **Service**（`--add-service`）  
2. **Port**（`--add-port`）  
3. **Rich rules**（細部條件）  
4. **`--direct`**：直接下 **iptables/nft** 語意，**不建議當預設**，難維護、順序敏感。  
5. **ipset**：常搭配 rich rule 或 direct，做大量位址集合。

## 3. 安裝與第一次執行

### 3.1 用 uv（開發／本機）

專案根目錄（若環境需代理，可先 `source ~/.proxy` 或使用 `./scripts/uv`）：

```bash
./scripts/uv sync --extra dev
./scripts/uv run fwctl -- --help
```

### 3.2 裝進目前 Python 環境

```bash
./scripts/uv pip install -e .
fwctl --help
```

### 3.3 確認 `firewall-cmd` 在 PATH

fwctl 會呼叫系統上的 `firewall-cmd`；若找不到，請安裝並啟動 **firewalld**。

## 4. 權限：Polkit、sudo、為何 `sudo fwctl` 會 command not found？

### 4.1 Authorization failed / polkit

若出現 **Authorization failed**、**polkit** 等訊息，代表 **PolicyKit 不允許目前使用者**透過 D-Bus 操作 firewalld，需以 **root** 執行（例如 `sudo`）。

### 4.2 sudo 與 PATH

`sudo` 常會 **重設 PATH**，因此 **venv 裡的 `fwctl`** root 找不到，出現：

`sudo: fwctl: command not found`

**建議寫法**（在一般使用者 shell，先解析路徑再給 sudo）：

```bash
sudo "$(command -v fwctl)" status
```

或寫死 venv 路徑、或 `sudo env "PATH=$PATH" fwctl …`（視 `sudoers` 是否允許）、或將 `fwctl` 裝到 `/usr/local/bin` 等 **root 預設 PATH 也找得到** 的位置。

---

## 5. 建議工作流（安全、可重現）

1. **先查**：`status` → `zone show` → `service list` / `port list` / `rule list` / `direct rules`。  
2. **變更前先預覽**：變更類子命令盡量加 **`--dry-run`**，確認印出的 `firewall-cmd` 參數無誤。  
3. **腳本略過確認**：加 **`--yes`**（仍會遵守防呆旗標，例如 direct 的 `--accept-drop-risk`）。  
4. **改 permanent 後**：執行 **`fwctl reload`**，讓 runtime 載入永久設定。

## 6. 子命令教學摘要

### 6.1 總覽：`status`

```bash
sudo "$(command -v fwctl)" status
sudo "$(command -v fwctl)" status --all-zones
```

可看 daemon 狀態、預設 zone、active zones；`--all-zones` 較冗長。

### 6.2 Zone：`zone list` / `zone show`

```bash
sudo "$(command -v fwctl)" zone list
sudo "$(command -v fwctl)" zone show trusted
```

### 6.3 服務與埠

```bash
sudo "$(command -v fwctl)" service list
sudo "$(command -v fwctl)" port list
# 變更範例（永久 + zone）
sudo "$(command -v fwctl)" service add http --zone public --permanent --yes
```

`list` 可加 **`--permanent`** 看永久檔內容。

### 6.4 Rich rules

```bash
sudo "$(command -v fwctl)" rule list
sudo "$(command -v fwctl)" rule add --rule 'rule family="ipv4" …' --permanent --yes
```

也可用 **`--rule-file`** 從檔案讀規則字串。

### 6.5 ipset

```bash
sudo "$(command -v fwctl)" ipset list
sudo "$(command -v fwctl)" ipset show MYSET --permanent
```

`show` 預設會再列出 entries；只要 info 可加 **`--no-entries`**。

**新增／刪除條目**（對應 `firewall-cmd --ipset=… --add-entry`／`--remove-entry`；變更類同樣可用 `--dry-run`、`--yes`）：

```bash
sudo "$(command -v fwctl)" ipset add-entry MYSET 10.0.0.1 --permanent --yes
sudo "$(command -v fwctl)" ipset remove-entry MYSET 10.0.0.1 --permanent --yes
```

- **`--verify-absent`**（僅 `add-entry`）：若 `query-entry` 已存在則拒絕，避免重複加入。  
- **`remove-entry`** 預設會先做 **`query-entry`**；不存在則拒絕（可用 **`--no-verify-present`** 略過，不建議）。  
- 若 ipset **名稱含 `ssh`**（不分大小寫），`remove-entry` 須再加 **`--accept-ssh-ipset-risk`**，避免誤刪 SSH 白名單條目。

**互動精靈**（與 `direct wizard-*` 類似：先 **dry-run** 預覽，再問是否實際執行；`--yes` 只略過最後一步確認）：

```bash
sudo "$(command -v fwctl)" ipset wizard-add
sudo "$(command -v fwctl)" ipset wizard-remove
```

- **wizard-add**：先選 **1** 建立**新空 ipset**（`--new-ipset` + `--type`，一律 permanent；可選多行 `--option`；建立後可選是否立刻加第一筆條目），或選 **2** 對**現有** ipset **加條目**（選 **2** 後會先印出 runtime／permanent 的 **ipset 名稱表**，再查 `--info-ipset` 是否存在）。其餘步驟含 **permanent**／**verify-absent**、dry-run 預覽。  
- **wizard-remove**：先印 **runtime／permanent 的 ipset 名稱表**；輸入名稱後再以 **`--get-entries`** 列出**條目**，可選編號或改手動輸入。名稱含 **ssh** 時須輸入大寫 **`SSH-IPSET-RISK`** 才可繼續（等同 CLI 的 `--accept-ssh-ipset-risk` 意圖）。

### 6.6 direct：僅檢視

```bash
sudo "$(command -v fwctl)" direct rules
sudo "$(command -v fwctl)" direct chains
sudo "$(command -v fwctl)" direct passthroughs
```

可加 **`--permanent`** 對齊永久設定。

## 7. direct 新增／刪除（防呆教學）

direct 錯誤順序或誤刪 **ACCEPT**，可能導致 **SSH 進不去**。請固定依下列順序操作。

### 7.1 從現有規則複製「可刪／可再加」的字串

1. 執行：`fwctl direct rules`（或加 `--permanent`）。  
2. 每一行格式大致為：  
   `ipv4 filter INPUT 110 -p tcp -m state --state ESTABLISHED -j ACCEPT`  
3. **`--rule` 要填的是「priority 後面整段」**，與 `firewall-cmd` 的 `--add-rule` 一致，例如：  
   `-p tcp -m state --state ESTABLISHED -j ACCEPT`  
   （前面的 `ipv4 filter INPUT 110` 已拆成 `--family` / `--table` / `--chain` / `--priority`。）

### 7.2 必走流程：dry-run → 再正式

```bash
# 1) 只預覽，不寫入
fwctl direct add --chain INPUT --priority 150 --rule "-s 10.0.0.0/8 -j ACCEPT" --dry-run

# 2) 確認無誤後（需 root）
sudo "$(command -v fwctl)" direct add --chain INPUT --priority 150 --rule "-s 10.0.0.0/8 -j ACCEPT" --yes
```

### 7.3 防呆旗標（務必讀）

| 情境 | 必要／建議 |
|------|------------|
| 規則含 **`-j DROP` / `-j REJECT`** | 必須加 **`--accept-drop-risk`**，且預設會再確認（可用 `--yes`）。 |
| 只允許 **filter + INPUT/OUTPUT/FORWARD** | 預設如此；若要 **nat / PREROUTING** 等，加 **`--allow-unusual-chain`**。 |
| **刪除** | 預設 **`--verify-present`**：先 `--query-rule`，不存在則拒刪。 |
| 刪除內容與 **SSH（22、ssh 字樣）** 有關 | 須 **`--accept-ssh-rule-risk`**。 |
| 避免重複新增 | 可加 **`--verify-absent`**（已存在則拒加）。 |
| 長規則 | 用 **`--rule-file`**。 |

### 7.4 刪除範例

```bash
fwctl direct remove --chain INPUT --priority 150 --rule "-s 10.0.0.0/8 -j ACCEPT" --dry-run
sudo "$(command -v fwctl)" direct remove --chain INPUT --priority 150 --rule "-s 10.0.0.0/8 -j ACCEPT" --yes
```

若 `verify-present` 失敗，請逐字對照 `direct rules` 輸出（含空白與逗號）。

### 7.5 互動式精靈（推薦不熟參數時使用）

與 `direct add` / `direct remove` **同一套防呆**，但改為逐步問答，並**固定先 dry-run 再問是否實際執行**：

```bash
sudo "$(command -v fwctl)" direct wizard-add
sudo "$(command -v fwctl)" direct wizard-remove
```

- **wizard-add**：問 family / table / chain / priority、rule 文字或檔案、是否 permanent、是否 `verify-absent`；若偵測到 **DROP/REJECT**，必須輸入大寫 **`DROP-RISK`** 才能繼續。  
- **wizard-remove**：可選 **模式 1** 貼上 `direct rules` 的**一整行**（程式會解析 family/table/chain/priority/rule），或 **模式 2** 分欄輸入；若規則疑似與 **SSH** 有關，必須輸入大寫 **`SSH-RISK`**。  
- 兩者結束前都會顯示 **dry-run** 的 `firewall-cmd` 列；最後再問是否寫入。腳本可再加 **`--yes`** 略過「最後一次是否執行」的確認（**不會**略過 DROP／SSH 的打字關卡）。

## 8. `--offline` 模式

當 **firewalld 未跑**或要在 **chroot／映像** 改檔案時，**全域選項 `--offline` 必須寫在子命令前**：

```bash
fwctl --offline status
fwctl --offline zone show public
```

- **沒有** runtime／D-Bus：`status` 不顯示 daemon state 與 active zones 的語意與線上相同。  
- **`reload` / `panic`**：fwctl 會拒絕（僅線上有意義）。  
- 傳給 offline 後端的參數會 **自動去掉 `--permanent`**（離線本質上就是改磁碟）。

## 9. 疑難排解（簡表）

| 現象 | 可能原因 | 可先做 |
|------|----------|--------|
| Authorization failed | Polkit 不允許一般使用者 | 用 `sudo "$(command -v fwctl)" …` |
| sudo: fwctl: command not found | sudo 縮 PATH | `sudo "$(command -v fwctl)"` 或固定絕對路徑 |
| SSH 突然進不去 | direct 順序／誤刪 ACCEPT、或來源不在 ipset 白名單、或 conntrack 滿 | 從 console／管理網段查；檢查 `direct rules` 與 `ssh-in` 類 ipset |
| 弱掃「全黑」 | 預設 DROP、掃描被封鎖 | 多為預期；要掃通需另開維護窗／掃描源白名單 |
