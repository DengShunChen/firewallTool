# 防火牆設定視覺化（fwctl viz）

## 視覺化對象與使用情境

| 情境 | 說明 |
|------|------|
| **一次性報告** | 變更前後執行 `fwctl viz export`／`html`／`markdown`，將 JSON／HTML／MD 附在變更單或工單。 |
| **本機儀表板原料** | JSON 含固定 `schema_version`，可由內部網頁或 BI 工具讀取繪圖（本專案另附靜態 HTML 範本）。 |
| **CLI 延伸** | 不需另開專案即可從現有 `firewall-cmd` 管線產出機讀快照。 |

**離線（`fwctl --offline`）**：僅能讀取磁碟上的設定，等同單一「永久」視角；**不提供** runtime 與 permanent 兩套對照，JSON 內 `drift_available` 為 `false`，HTML／Markdown 的 drift 區塊會顯示說明文字。

**網路**：`viz html` 產生的頁面會自 **jsDelivr CDN** 載入 Mermaid；無對外網路時可改用 **`viz markdown`**（圖表為 **Mermaid fenced code block**，GitHub／GitLab 等可預覽）或僅使用 `viz export` 的 JSON。

## 指令

```bash
# JSON 印到 stdout
sudo "$(command -v fwctl)" viz export

# JSON 寫檔（可加上原始 list-all-zones 文字）
sudo "$(command -v fwctl)" viz export -o /tmp/fw-snap.json --include-raw

# 即時查詢並產 HTML
sudo "$(command -v fwctl)" viz html -o /tmp/fw-report.html

# 由既有 JSON 產 HTML（不需再查 firewalld）
fwctl viz html --input /tmp/fw-snap.json -o /tmp/fw-report.html

# 即時查詢並產 Markdown（Mermaid + 表格，可貼 PR／Wiki）
sudo "$(command -v fwctl)" viz markdown -o /tmp/fw-report.md

# 由既有 JSON 產 Markdown
fwctl viz markdown --input /tmp/fw-snap.json -o /tmp/fw-report.md
```

讀取 firewalld 時權限與一般 `fwctl` 相同；若 PolicyKit 拒絕非 root，請用 root 執行。

## JSON schema（`schema_version`: **2**）

頂層欄位：

| 欄位 | 型別 | 說明 |
|------|------|------|
| `schema_version` | int | 目前為 **2**（**1** 僅名稱級 ipset／無 `direct_rules_parsed`）。 |
| `generated_at` | string | UTC ISO8601 時間戳。 |
| `backend` | string | `firewall-cmd` 或 `firewall-offline-cmd`。 |
| `default_zone` | string | `--get-default-zone`。 |
| `runtime` | object \| null | 線上模式：`{ "zones": [...] }`，可選 `raw_list_all_zones`；offline 為 `null`。 |
| `permanent` | object | `{ "zones": [...] }`；offline 時僅此側有資料。 |
| `drift_available` | bool | 是否可比對兩套設定。 |
| `drift` | object | 線上：`{ "zones": { ZONE_NAME: { "lists", "scalars", "rich_rules" } } }`；offline 為說明用物件。 |
| `ipsets` | object | 見下節。 |
| `direct_rules` | string[] | `--direct --get-all-rules` 逐行；失敗時可能為 `[]`。 |
| `direct_rules_parsed` | object[] | 每行解析結果（`family`、`table`、`chain`、`priority`、`tokens` 或 `parse_error`）。 |
| `direct_allow_matrix` | object | 由 `direct_rules_parsed`（僅 `-j ACCEPT`）與 `ipsets.details` 對照產生；鍵 `input`／`output`／`forward`／`other` 為列陣列，`stats` 為略過筆數統計，`note` 為免責說明。每列含 `sources`、`destinations`、`proto`、`dports`、`sports`、`in_interfaces`、`out_interfaces`、`ipset_matches`、`ipset_resolved`（含 `entries_summary`）、`raw` 等。 |

### `ipsets`（schema 2）

| 子鍵 | 型別 | 說明 |
|------|------|------|
| `runtime` / `permanent` | string[] | `--get-ipsets`／`--permanent --get-ipsets` 名稱列表（與 schema 1 相同）。 |
| `details` | object | `{ "runtime": [...], "permanent": [...] }`，每筆為單一 ipset：`name`、`info`（`--info-ipset`，可能截斷）、`entries`（`--get-entries` 前 N 筆原始字串）、`entries_total`、`entries_truncated`；另含 **濃縮**：`entries_compact`（同一 /24 內多主機寫成 `172.16.90.[1,2]`，連續三個以上末段寫成 `10.0.0.[0-255]`；單一主機仍為四段式；非 /32 的 IPv4 CIDR 與 IPv6 仍用 `collapse_addresses`）、`entries_compact_total`、`entries_summary`、`entries_collapse_input_capped`（條目過多時僅以前 20000 筆濃縮）；`info_error`／`entries_error`（可選）。 |

### `runtime`／`permanent` 內 `zones[]` 單一 zone

| 欄位 | 型別 | 說明 |
|------|------|------|
| `name` | string | zone 名稱。 |
| `active` | bool | 標頭是否含 `(active)`（僅 list-all-zones 語意）。 |
| `attributes` | object | 來自 `list-all-zones` 的鍵值；`services`、`ports`、`interfaces` 等為 **字串陣列**，其餘多為字串。 |
| `rich_rules` | string[] | 該 zone 的 rich rule 行（已 trim）。 |

### `drift.zones[ZONE].lists`（可比對集合）

各鍵（如 `services`、`ports`）之值：

```json
{
  "only_runtime": ["..."],
  "only_permanent": ["..."],
  "both": ["..."]
}
```

### `drift.zones[ZONE].scalars`

鍵如 `target`、`masquerade`：`{ "runtime": "...", "permanent": "...", "match": true|false }`。

## HTML 報告內容

1. **Mermaid flowchart**：預設 zone 連到各 zone 節點，標註 active 與介面列表（摘要）。  
2. **Drift 表**：各 zone 的 services／ports 在 runtime-only、permanent-only 的差異。  
3. **ipset**：名稱兩行摘要；**Mermaid** 顯示「僅 runtime／交集／僅 permanent」數量；**表格**列出各 ipset 的 `info` 摘要、條目數、**濃縮預覽**（同一 /24 主機為 `a.b.c.[1,2]` 或連續末段 `a.b.c.[0-255]`；CIDR／IPv6 另做合併）與可展開的完整 `info`（來自 schema 2 的 `ipsets.details`；舊 JSON 會提示無詳情）。  
4. **Direct 允許摘要**：**Mermaid pie**（各 chain 的 ACCEPT 列數）；**表格**依 INPUT／OUTPUT／FORWARD／其它列出掃描到的來源／目的／協定／埠／介面／ipset 名稱與**對照本快照的 ipset 條目摘要**（最佳 effort，不含 zone／rich rules）。  
5. **direct 全規則**：**Mermaid** 依 family／table／chain 分 subgraph，組內依 priority 串接（僅輔助閱讀，**不**代表實際 netfilter 順序）；**表格**列出解析後欄位；**摺疊區**保留原始 `--get-all-rules` 行。

## Markdown 報告內容（`viz markdown`）

與上列 HTML **章節對齊**（拓樸、drift、ipset、Direct 允許摘要、direct 全表）；差異為 **Markdown 表格**、圖表皆在 **fenced code block**（語法標為 `mermaid`，不需 jsDelivr）、原始 direct 行以 **fenced code block**（語法標為 `text`）列出（無摺疊 UI）。
